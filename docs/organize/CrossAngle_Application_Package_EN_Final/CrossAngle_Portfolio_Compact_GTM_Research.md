내가 크로스앵글에서 바로 기여할 수 있는 방식
GTM과 Research를 따로가 아니라, 서로 연결된 실행 구조로 봅니다.
GTM Team에 맞는 기여 방식
• 시장의 복잡한 흐름을 짧고 읽기 쉬운 콘텐츠 단위로 정리
• 프로젝트별 핵심 메시지와 업데이트 포인트를 문서화
• 채널별 포맷 차이를 고려해 재사용 가능한 제작 규칙 정리
• 피드백과 운영 질문을 다음 산출물 개선으로 연결
Research Team에 맞는 기여 방식
• 참여/관찰 데이터를 구조적 메모로 전환
• 프로젝트·시장 이슈를 조사–정리–요약–초안 작성까지 연결
• 규제·운영·사용자 관점을 한 문서 안에서 함께 다룸
• 불확실한 내용은 과장하지 않고 fact / assumption을 분리
Work style | 문제 정의 → 근거 수집 → 구조화 → 짧은 메모 → 피드백 반영 → 다음 실행
