# Ariadne 정리본

## 1) 시장 / 문제 정의 — 3줄 요약
- 패션 쇼핑에서 문제는 `무엇을 볼지`보다 `후보군 중 무엇을 살지 결정하는 순간`에 더 크게 발생한다.
- 사용자는 여러 패션 플랫폼과 브랜드몰을 교차 이용하지만, 각 장바구니를 한 번에 비교할 수 없어 캡처·DM·혼자 고민에 의존한다.
- 그래서 Ariadne는 `취향이 비슷한 사람의 빠른 판단`과 `통합 저장소`로 장바구니 단계의 결정 비용을 줄이는 서비스로 간다.

## 2) 왜 이 방향으로 갔는가 — 핵심 판단 3개
1. **탐색보다 결정 단계가 더 선명한 pain point다.**  
   추천/랭킹은 이미 많지만, 마지막 한 벌을 고르는 문제는 아직 비어 있다.

2. **결제는 외부 플랫폼에서 하고, 우리는 고민 해결 레이어에 집중한다.**  
   무신사·지그재그·브랜드몰과 경쟁하지 않고 그 위에 붙는 구조가 진입도 쉽고 설명도 명확하다.

3. **단일 플랫폼 기능이 아니라 멀티 플랫폼 저장소가 차별점이다.**  
   기존 ‘골라줘’류 기능이 자사 상품 비교에 머무는 반면, Ariadne는 여러 구매처 후보를 한곳에 모으는 것이 핵심이다.

## 3) 핵심 사용자 흐름
1. **취향 입력**  
   무드 / 좋아하는 아이템 / 구매 시 중요 요소를 받아 유사 취향 피드를 정렬한다.

2. **후보 수집**  
   사용자는 여러 쇼핑앱·브랜드몰에서 마음에 드는 상품을 저장하거나 캡처한다.

3. **저장소 정리**  
   후보군을 폴더 / 태그 / 카테고리 기준으로 정리해 통합 장바구니처럼 모은다.

4. **투표 질문 작성**  
   후보 2~5개와 상황 태그(예: 출근, 데이트, 겨울)를 붙여 질문한다. 필요하면 본인 사진도 optional로 추가한다.

5. **유사 취향 피드백 획득**  
   비슷한 취향의 유저가 투표 / 댓글 / 저장으로 판단 근거를 준다.

6. **결정 후 원 판매처 이동**  
   결과를 보고 최종 선택한 뒤, 실제 구매는 원래 플랫폼 또는 브랜드몰에서 진행한다.

## 6) Figma 화면 가이드 — 4장

### Screen 01. 취향 세분화
**목적**
- 가입 직후 유저의 스타일 축을 빠르게 잡는 화면.

**핵심 컴포넌트**
- 상단 타이틀: `좋아하는 무드를 골라주세요`
- 무드 칩 8~12개
- 좋아하는 아이템 카드 3~4개
- 구매 시 중요 요소 칩(핏 / 가격 / 소재 / 활용도)
- CTA: `다음`

**포인트**
- 이 화면은 예쁘기보다 빠르게 선택되게 해야 함.
- 선택 결과가 이후 피드 정렬의 기준이라는 메시지가 필요함.

---

### Screen 02. 메인 피드 (투표 카드)
**목적**
- 질문을 카드 한 장 안에서 읽고 바로 투표하게 만드는 화면.

**핵심 컴포넌트**
- 작성자 + 취향 라벨
- 질문 문장 1줄
- 후보 2~3개 타일
- 버튼: `투표하기` / `저장`
- 태그: `#출근 #겨울 #미니멀`
- 하단 보조 카드: 추천/공유 콘텐츠 한 장

**포인트**
- 피드의 기본 단위는 “게시글”이 아니라 “즉시 판단 가능한 카드”여야 함.
- 저장은 투표 다음 액션으로 붙는 구조가 자연스럽다.

---

### Screen 03. 투표 작성
**목적**
- 후보군을 구조화해서 질문하게 만드는 화면.

**핵심 컴포넌트**
- 상단 소스 선택: `저장소에서 불러오기 / 앨범`
- 후보 슬롯 2~5개
- 질문 입력창
- 태그 입력칩
- optional self-photo 토글
- CTA: `게시하기`

**포인트**
- 작성 flow는 복잡해 보이면 안 되고, “후보 추가 → 질문 입력 → 태그 지정” 3단계가 명확해야 함.
- 저장소가 이 화면의 source가 되도록 연결감을 줘야 함.

---

### Screen 04. 저장소
**목적**
- 여러 앱에서 모은 후보를 장바구니처럼 정리하는 화면.

**핵심 컴포넌트**
- 검색 바
- 폴더 카드(예: 겨울 코트 / 출근룩 / 데이트)
- 필터 칩(전체 / 상의 / 하의 / 아우터 / 신발)
- 저장된 상품 그리드
- 구매 링크 / 바로가기

**포인트**
- 저장소는 단순 찜 목록이 아니라 `비교용 후보군 공간`으로 보여야 함.
- 추후에는 원 판매처 링크와 연결되는 통합 장바구니로 확장 가능해야 함.

## 7) 바로 복붙해서 쓸 수 있는 섹션 제목
- `패션 쇼핑의 마지막 선택을 해결하는 SNS`
- `추천 이후, 결정은 아직 비어 있다`
- `투표 + 저장소로 장바구니 의사결정을 줄인다`
- `결제는 밖에서, 판단은 여기서`

## 8) 한 줄 소개 후보
- **여러 쇼핑앱의 후보군을 모아, 비슷한 취향의 사람들과 빠르게 고르는 패션 결정 SNS**
- **장바구니 단계의 고민을 투표와 저장소로 해결하는 패션 SNS**
